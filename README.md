# MyMem

> 面向 OpenClaw 个人助理的强大长期记忆系统。

MyMem 让 OpenClaw 智能体真正拥有跨会话、可检索、会治理、会反思、会自我改进的长期记忆。它会记住用户的生活偏好、工作背景、人际关系、长期目标、重要安排、关键决策、成功经验、失败教训和可复用习惯，并在之后的对话中自动带回最相关的上下文。

它不是一个简单的笔记本，也不是把聊天记录原样塞进数据库。MyMem 是一套完整的记忆系统：自动捕捉、智能提取、混合检索、上下文注入、反思沉淀、噪声过滤、生命周期维护、错误学习和记忆治理都在同一个闭环里工作。

---

## 为什么需要 MyMem

普通智能体最大的问题之一是失忆。

每次新会话都要重新解释自己的偏好、日程背景、家庭情况、工作重点、正在推进的事项和之前已经说过的结论；复杂事情中已经犯过的错误，下次还可能再犯；不同助理能力之间也容易缺少稳定的共享上下文。

MyMem 解决的是这个问题：让智能体随着使用时间变长，越来越理解你、你的生活方式、工作节奏和长期偏好。

它不会把所有内容都机械保存下来。它会尽量留下未来有用的信息，过滤低价值噪声，更新过时事实，降低错误记忆的影响，并把真实工作中形成的经验变成可复用知识。

---

## 核心能力

MyMem 的核心体验是自动完成记忆系统该做的事，而不是让用户手动维护一堆笔记。

- 用户的表达偏好、生活习惯、饮食偏好和沟通方式
- 家人、朋友、同事、客户和常联系对象的关键信息
- 正在推进的工作、学习、旅行、家庭和个人计划
- 做过的重要决定以及背后的原因
- 常用服务、工具、地点、商家、团队和协作对象
- 发生过的问题、处理方式、复盘结论和下次提醒
- 多次成功实践中沉淀出的个人方法和工作习惯

它会把这些信息提炼成结构化长期记忆，自动召回相关上下文，按助理和场景隔离记忆边界，在会话结束时做反思沉淀，并把用户纠正、工具失败、成功案例和坏召回反馈纳入自我改进闭环。

记忆数据保存在用户自己的 OpenClaw 环境中。MyMem 关注的是对未来有价值的长期记忆，而不是完整复刻用户所有输入。

## 记忆架构设计

MyMem 真正强的地方不只是“能存”，而是它把长期记忆当成一个会演化的系统来设计。

### 六类记忆模型

MyMem 使用 6 类记忆来组织个人助理的长期上下文：

- `profile`：用户画像，例如身份信息、长期目标、生活方式和稳定背景
- `preferences`：偏好和约束，例如沟通方式、饮食偏好、工作习惯、工具选择
- `entities`：关键实体，例如家人朋友、同事客户、地点、服务、项目、设备和账号体系
- `events`：发生过的事件，例如一次安排、一次沟通、一次旅行、一次事故或一次阶段进展
- `cases`：具体案例，例如某次问题如何处理、某个任务如何完成、某个失败如何修复
- `patterns`：可复用模式，例如反复有效的方法、协作规则、避坑经验和自我改进结论

这不是简单标签。不同类别有不同的治理策略：画像会持续合并，偏好和实体支持随时间替换，事件和案例保持轨迹记录，模式会被不断提炼成更稳定的做事方法。

### OpenClaw 工具 category 兼容层

OpenClaw 工具参数里的 `category` 是兼容旧版 memory 插件和存储层的分类，不是 MyMem 智能提取器内部的完整 6 类模型。工具层仍接受：

- `preference`
- `fact`
- `decision`
- `entity`
- `reflection`
- `other`

MyMem 会在写入时保留这个顶层 category 以兼容 OpenClaw，同时在 metadata 里写入真正的智能记忆分类 `memory_category`。智能提取和召回展示优先使用 `memory_category`。

常见映射关系是：

- `profile` -> `fact`
- `preferences` -> `preference`
- `entities` -> `entity`
- `events` -> `decision`
- `cases` -> `fact`
- `patterns` -> `other`

因此你不会在工具参数里看到 `profile`、`events` 或 `patterns`。它们属于 MyMem 内部的 smart memory category；工具参数里的 `fact`、`decision`、`other` 会被 MyMem 进一步映射和补充。`reflection` 是反思记录的兼容存储类型，反思内容通常会作为 `patterns` 参与长期学习。

### 知识和经验隔离

MyMem 把记忆分成两条逻辑通道：

- `knowledge`：相对稳定的知识，包括 `profile`、`preferences`、`entities`、`patterns`
- `experience`：发生过的经历，包括 `events`、`cases`

这让个人助理既能回答“我喜欢什么、我有哪些长期规则、某个人是谁”，也能回答“上次发生了什么、那次怎么处理、为什么后来换了做法”。

知识和经验不会混在一起互相污染。检索时 MyMem 会根据问题意图提升对应通道：问偏好时优先找稳定知识，问“上次”“之前”“那次”时优先找经验轨迹。生命周期上，稳定知识保留更久，旧经验会随着时间自然降权，但被反复使用或验证有效的经验可以升格成模式。

### 三层摘要结构

每条智能记忆都不是一段扁平文本，而是带有三层表达：

- `L0`：一句话索引，用于快速检索、去重和低成本召回
- `L1`：结构化概览，用于回答“这条记忆大概是什么”
- `L2`：完整叙事，用于需要细节、来源和上下文时展开

这让 MyMem 可以在不同场景下选择不同粒度。普通自动召回只需要带入很短的 L0/L1，复杂追问再展开 L2；检索时也会同时利用摘要、概览和全文，既快又不丢细节。

### 自动升降级和生命周期

MyMem 内部有两套层级：

- `tier`：`core`、`working`、`peripheral`
- `layer`：`durable`、`working`、`reflection`、`archive`

重要、常用、可信的记忆会从 `peripheral` 升到 `working`，再升到 `core`；低价值、久未使用、被证明无关或已经过期的记忆会降级或归档。这个过程会综合时间、访问次数、重要性、置信度、反馈信号和记忆类型，而不是只按创建时间粗暴删除。

所以 MyMem 的记忆库不是越积越臃肿。它会把真正重要的长期知识留在前台，把临时经验放在工作层，把会话反思放进反思层，把过时或低价值内容移到归档层。个人助理看到的是经过治理后的上下文，而不是原始聊天垃圾堆。

### 冲突、替换和去重

现实中的记忆会变化：用户偏好会变，计划会取消，某个联系人信息会更新，过去有效的方法也可能过期。

MyMem 对这些变化有专门处理：

- 相似记忆会先做向量预筛，再做语义去重
- 重复内容会跳过或合并
- 新事实可以替换旧事实，并保留 `supersedes` / `superseded_by` 关系
- 冲突记忆会被标记、降权或归档
- 支持性证据会强化已有记忆，而不是制造重复条目
- 相似旧记忆会被压缩成更高质量的合并记忆

这意味着 MyMem 不是只会“记住更多”，而是会持续把记忆整理得更准确、更干净、更可用。

### 面向召回的检索链路

MyMem 的召回不是简单向量搜索。它会结合：

- 向量语义检索
- 关键词和 BM25 命中
- L0/L1/L2 多粒度文本
- 类别意图识别
- 知识/经验通道提升
- 时间衰减和访问强化
- 重要性、置信度和生命周期状态
- 重排序模型二次排序

最后注入上下文前，还会控制体积、去掉噪声、避免重复注入，并把记忆当成历史背景而不是新指令。用户看到的是“助理刚好想起来了关键内容”，背后其实是一整条记忆工程流水线。

## 使用体验

安装 MyMem 后，只需要在 OpenClaw 中提供必要的模型凭据，然后正常使用智能体。

MyMem 的默认配置已经按实际使用场景调好。大多数用户不需要理解检索权重、生命周期策略、注入预算、反思策略或内部开关，也不需要手动维护记忆。

你会逐渐感受到：

- 智能体更清楚你喜欢怎样沟通
- 新会话也能延续生活和工作背景
- 之前踩过的坑更少重复
- 已经做过的决策不会轻易丢失
- 多个助理能力之间上下文更稳定
- 成功经验会沉淀为之后可复用的方法

## MyMem 可以形成的记忆

- 用户喜欢直接、简洁、先给结论再展开的回答。
- 用户周末通常不安排高强度工作，适合处理生活计划和轻量复盘。
- 用户偏好的餐厅、咖啡、出行方式、购物平台和常用服务。
- 家人、朋友、同事或客户的重要偏好、称呼、纪念日和协作方式。
- 某个长期目标的阶段进展、阻碍和下一步计划。
- 某类问题已经有成熟处理方式，下次应直接复用。
- 消息平台的包装元数据不应该进入长期记忆。

这些记忆会让智能体更像一个长期陪伴的个人助理，而不是每次从零开始的问答模型。

## 为强默认配置而设计

MyMem 的产品体验不是让用户研究大量开关，而是默认就好用。

捕捉、提取、召回、反思、治理和上下文控制已经组合成默认策略。用户启用后正常使用，记忆系统会在后台持续运行。

高级选项存在，是为了特殊部署和深度调优；普通用户只需要安装、启用、正常使用。

## 适合哪些场景

MyMem 特别适合：

- 长期使用 OpenClaw 的个人或团队
- 经常开启新会话但希望保留连续上下文
- 希望助理同时理解生活和工作上下文的用户
- 需要长期跟踪计划、习惯、关系和阶段目标的人
- 希望智能体记住沟通方式、协作偏好和个人约束
- 使用多个专用助理能力的工作流
- 想把错误、修复和成功经验沉淀下来
- 不想手动维护个人记忆文档的用户

如果你希望智能体越用越懂你的工作，MyMem 就是为这个目标设计的。

## 安装与启用

把本仓库链接发给你的 OpenClaw，从仓库克隆到 /root/.openclaw/workspace/plugins/mymem 安装。

MyMem 作为 OpenClaw 的 memory 插件，需要在 OpenClaw 设置中填入下方的推荐配置。

启用后，MyMem 会自动开始捕捉、治理和召回长期记忆。

---

## 推荐配置

推荐只配置模型连接信息和插件挂载关系，其它能力交给 MyMem 内置策略处理。下面这段配置形态适合把 MyMem 作为长期记忆插件使用：

```json
{
  "plugins": {
    "load": {
      "paths": [
        "/root/.openclaw/workspace/plugins/mymem"
      ]
    },
    "allow": [
      "mymem"
    ],
    "slots": {
      "memory": "mymem"
    },
    "entries": {
      "mymem": {
        "enabled": true,
        "hooks": {
          "allowConversationAccess": true
        },
        "config": {
          "embedding": {
            "baseURL": "https://dashscope.aliyuncs.com/compatible-mode/v1",
            "apiKey": "${DASHSCOPE_API_KEY}",
            "model": "text-embedding-v4"
          },
          "llm": {
            "baseURL": "${LLM_API_URL}",
            "apiKey": "${LLM_API_KEY}",
            "model": "${LLM_API_MODEL}"
          },
          "retrieval": {
            "rerankEndpoint": "https://dashscope.aliyuncs.com/compatible-api/v1/reranks",
            "rerankApiKey": "${DASHSCOPE_API_KEY}",
            "rerankModel": "qwen3-rerank"
          }
        }
      }
    }
  }
}
```

`embedding` 负责把生活、工作、关系、偏好和计划转成可检索的长期记忆向量，

`llm` 负责智能提取和反思，

`retrieval` 负责在召回后重新排序，让个人助理更容易找到真正相关的记忆。

推荐的向量模型和重排模型都来自阿里云百炼，`DASHSCOPE_API_KEY` 从阿里云百炼控制台获取，建议通过环境变量注入。

如果你的 OpenClaw 环境还对工具做白名单控制，推荐允许 MyMem 的全部工具。核心工具包括 `mymem_recall`、`mymem_store`、`mymem_update`、`mymem_forget`；治理和诊断工具包括 `mymem_stats`、`mymem_list`、`mymem_debug`、`mymem_doctor`、`mymem_promote`、`mymem_archive`、`mymem_compact`、`mymem_explain_rank`；自我改进工具包括 `self_improvement_log`、`self_improvement_extract_skill`、`self_improvement_review`、`self_improvement_distill`。这组自我改进工具是 MyMem 的关键能力之一，它让个人助理不只会记住事实，还能从长期使用中沉淀偏好、经验和可复用的做事方式，也就是自进化，自动创建技能。

MyMem 的核心理念很简单：

> 记住能帮助未来工作的内容，过滤会制造噪声的内容，并从真实使用中持续变强。
